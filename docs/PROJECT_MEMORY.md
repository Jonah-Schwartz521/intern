## Decisions Made

- **Framework: Tauri 2.x.** Chosen over Electron. Intern is an always-resident hotkey/tray utility, the textbook Tauri-shaped product: lean footprint matters because it runs all day, and the Rust backend suits the audio/transcription work. Accepted tradeoff: to keep Rust exposure near zero, lean on official @tauri-apps plugins and the JS frontend, and treat "I need custom Rust" as a stop-and-ask signal. Windows-first sidesteps Tauri's main downside (per-OS webview rendering differences).
- **Name: Intern.** Checked for conflicting apps; no single product owns the name (InternLM is a model family, "AI intern" is common marketing copy but not a product). Fine for a personal tool. Logged so we do not revisit.
- **Scope boundaries.** Slack/Teams sending, mutual-availability scheduling, autonomous execution without confirmation, and live real-time streaming transcription are OUT of MVP. Email moved IN scope this session as draft-and-handoff (Intern creates a Graph draft, the user sends manually); autonomous email SENDING stays out. See the "Email drafting" decision below and NORTH_STAR.md.
- **Transcription order.** Start with file-based and playback-triggered transcription. Live streaming transcription is a later, bigger lift.
- **Architecture.** Single Claude call that routes intent to start. No internal sub-agents until the loop works. Orchestration lives in the frontend; reach into Rust only if a plugin genuinely cannot do the job.
- **LLM model routing + caching.** Use a smart model cost-effectively rather than picking a cheap-everything model. Default routine intent parsing to Haiku 4.5; escalate to Opus 4.8 only for ambiguous, multi-step, or context-heavy requests. Cache the static system prompt (instructions + tool definitions) so repeated input bills at the cache-read rate. Keep the model behind a thin abstraction (config value, not hardcoded). No Batch API (it is async, Intern is real-time). Estimated cost at ~50 commands/day: roughly $2/month routed, ~$16/month if everything went through Opus. Rationale: intent parsing is mostly classification/extraction, which the cheap tier handles well; Opus is reserved for the requests that justify it.
- **Transcription: local whisper.cpp.** Run transcription on-device via whisper.cpp instead of a paid per-minute API. Zero per-minute cost, works offline, private. Pipeline: ffmpeg extracts audio -> whisper.cpp transcribes -> text. Shelled out from Tauri like ffmpeg. Optional cloud fallback (e.g. Groq Whisper) can be added later, not MVP. This was the cost line item that actually mattered (audio is billed per minute), so removing it removes the only real recurring cost.

- **Hotkey: Ctrl+Shift+Space.** Changed from the originally-planned Ctrl+Shift+I because Ctrl+Shift+I collides with the webview DevTools shortcut in dev mode (pressing it opened DevTools instead of firing the global shortcut). Space is clean, one-handed, no collision. Registered via CmdOrCtrl+Shift+Space in code (CmdOrCtrl resolves to Ctrl on Windows). NOTE: update NORTH_STAR.md, which still lists Ctrl+Shift+I.
- **Outlook calendar + mail integration (Microsoft Graph, PKCE).** Provider is Outlook via Microsoft Graph. Auth is the PKCE authorization-code flow for a public native client (no client secret). Current app: client ID `f4c142b4-85f2-4d7d-801d-5e8c59fe0676`, registered **Personal Microsoft accounts only**, so the auth endpoint is `https://login.microsoftonline.com/consumers/oauth2/v2.0` (NOT `/common/`, which requires an "All" audience). Redirect `http://localhost` (Mobile/desktop platform, loopback, port-agnostic), public client flows enabled. Scopes: `Calendars.ReadWrite Mail.ReadWrite User.Read offline_access` (Mail.ReadWrite for creating drafts; NO Mail.Send, sending stays manual in Outlook). See the app-registration saga under Blockers/Gotchas for why the client ID / endpoint changed.
  - **Three plugins added** (touches Rust config, flagged and approved): `tauri-plugin-oauth` (community, FabianLars) spawns a localhost server to catch the redirect; `@tauri-apps/plugin-http` for CORS-free token + Graph calls; `@tauri-apps/plugin-store` for token storage. All registered in lib.rs, deps in Cargo.toml, capabilities scoped in default.json.
  - **Why plugin-http and not browser fetch for token/Graph:** the Azure app is a NATIVE client. Microsoft only returns CORS headers on the `/token` endpoint for the SPA platform type, so a webview fetch would be blocked. plugin-http runs the request through Rust and sidesteps CORS. (The Anthropic call stays on browser fetch; it works via the dangerous-direct-browser-access header.)
  - **Loopback port:** the oauth plugin binds a random localhost port; Azure ignores the port on `http://localhost` loopback redirects, so it matches the registered redirect. The `redirect_uri` sent to authorize and token must be identical (`http://localhost:{port}`).
  - **Capabilities:** used `oauth:allow-start` + `oauth:allow-cancel` (auto-generated per-command perms, guaranteed to exist) rather than `oauth:default`, since the community plugin may not define a default set. `http:default` is URL-scoped to `login.microsoftonline.com/*` and `graph.microsoft.com/*` only.
  - **Could not compile-check Rust in the agent environment:** crates.io was unreachable (SSL reset) so the three new crates could not be fetched. Frontend `tsc` passes. First `npm run tauri dev` on a normal network downloads the crates and validates capabilities.
- **Transcription: Const-me/Whisper (GPU), no ffmpeg, everything bundled.** File transcription shells out to a bundled binary (no whisper-rs crate, keeping Rust near zero).
  - **Whisper engine: Const-me/Whisper**, a Windows-only fork of whisper.cpp that runs on the GPU via DirectCompute/Direct3D 11 (works on most GPUs, not just NVIDIA). Chosen for GPU speed on a Windows-first app. Pinned to release **1.12.0 (2023-07-22, its last release, stale but functional)**. The `cli.zip` asset contains `main.exe` (0.2 MB) + `Whisper.dll` (0.55 MB); the DLL must sit next to the exe. CLI is whisper.cpp-style: `-m <model>`, `-f <input>`, `-otxt` (writes `<input>.txt`, there is NO `-of`), `-nt`, `-l`, plus `-la` (list GPU adapters) and `-gpu`. Model format is standard ggml, so `ggml-base.bin` works.
  - **Accuracy note for future sessions:** the stated rationale was "upstream removed direct binary downloads," but Stage 0 verification found ggml-org/whisper.cpp DOES still ship Windows prebuilts (v1.9.1, `whisper-bin-x64.zip`, with a `whisper-cli.exe`). Const-me was chosen anyway for GPU acceleration; recorded so we don't re-derive this.
  - **ffmpeg: DROPPED (Stage 3 verified not needed).** Const-me `main.exe` decodes raw `.wav`, `.mp3`, AND `.mp4` (video, audio track pulled) directly via Windows Media Foundation. Test: all three transcribed with exit 0, each logging "Created source reader from the file ...". So there is NO ffmpeg conversion step and NO ffmpeg bundle (~80 MB and the gyan `.7z`-extraction problem both avoided). Revisit only if some future input format fails MF decode.
  - **Transcript capture: STDOUT.** `main.exe -m <model> -f <input> -nt` prints the clean transcript to stdout; all GPU/perf logging goes to stderr. Capture `Command.execute().stdout` — no `-otxt`, no temp output file. Verified: jfk.wav → "And so my fellow Americans ask not what your country can do for you, ask what you can do for your country."
  - **Perf (Intel Arc 140T GPU):** an 11 s clip ran in ~1.1 s total (0.5 s model load + 0.6 s GPU inference). The "30-60 s for 10 min" caveat was a CPU-era worst case; GPU is far faster.
  - **Model: `ggml-base.bin`** (~142 MB, multilingual) is BUNDLED as a Tauri resource, NOT downloaded. First-run download was built and then abandoned: Hugging Face TLS-reset intermittently on the target network across BOTH `curl` AND PowerShell `Invoke-WebRequest`. The reset is network-level, so the curl->IWR swap was a dead end. Bundling is deterministic and offline (installer bloat accepted). Copied from the resource into app-data on first use with an exact-byte-size integrity guard (147,951,465 bytes) so a truncated copy can never poison later runs.
  - **Binaries bundled as resources, resolved via the `$RESOURCE` scope variable (NOT JS `resolveResource`).** `main.exe` + `Whisper.dll` (~0.75 MB, co-located in `src-tauri/resources/whisper/`, committed) plus the model ship via `bundle.resources`. The `whisper-run` shell scope `cmd` is `$RESOURCE/resources/whisper/main.exe` - Tauri v2 shell scope supports the `$RESOURCE` variable, and `Command.create(name)` matches the scope by name and executes its `cmd`, so path resolution belongs in the scope, not JS. Whisper.dll co-locates with the exe in the same resource folder. Works in dev and installed builds.
  - **plugin-dialog** (file picker for Transcribe file) was installed, reverted, then re-added for the wiring stage. Now in place.
- **Voice input records `audio/mp4`, not MediaRecorder's default `webm/opus`.** Mic capture uses `getUserMedia` + `MediaRecorder` with an explicit `audio/mp4` mimeType. Reason: whisper's decoder is Windows Media Foundation, which decodes AAC (in mp4) but NOT WebM/Opus. Recording mp4/AAC means MF reads the file directly with no converter step (falls back to the browser default only if a machine cannot record mp4, and warns). Verified in Stage 2: `MediaRecorder.isTypeSupported("audio/mp4")` is true on this machine and the mp4 clip transcribed. Voice transcripts go to the input box (a command to review and send); an empty/too-short capture (< 2 KB) is skipped before hitting whisper.
- **System audio capture via Stereo Mix.** "Transcribe what's playing" taps the Windows Stereo Mix loopback device: enumerate audio inputs, match `/stereo mix/i`, pass its browser `deviceId` to `getUserMedia({ audio: { deviceId } })`, then the same mp4 -> whisper path. This is the clean digital loopback (works with the system muted or on headphones), unlike an acoustic mic pickup. System-audio transcripts render as a transcript bubble (content), while mic transcripts go to the input box (command). CAVEAT: Stereo Mix ships DISABLED by default on most Windows machines and must be enabled manually in Sound settings - fine for personal use, a blocker for distribution without a manual-enable step.
- **Email drafting (moved IN scope; was NOT-MVP).** Intern drafts and CREATES an Outlook draft via Graph (`POST /me/messages`, Mail.ReadWrite), then opens it via its `webLink` in browser Outlook. Deliberately NO `Mail.Send` - draft-and-handoff, the user reviews and sends manually. UX is an inline prefilled compose card ("Model B"): the `draft_email` tool has Claude generate To + a specific Subject + a full first-pass Body up front, the card renders in the chat stream (not a modal) with all three editable, and Create draft posts the edited field values. Edit-don't-interrogate: always draft, never ask field by field; only ask first if the recipient is genuinely unresolvable (then still draft subject+body and leave To blank). Single natural-language path - an earlier blank "Draft email" overflow-menu button was added then removed because a requestless button can only open an empty card. Email-drafting requests escalate to Opus in `pickModel` because Haiku under-fills bodies. (Graph draft code in `src/mail.ts`.)
- **Dev environment: Windows-native only.** Project lives at C:\dev\intern. A duplicate copy existed under WSL (\\wsl.localhost\Ubuntu\...) and caused repeated failures: Windows npm choking on Linux paths (EISDIR/EPERM), CMD rejecting UNC paths, stale edits landing in the wrong copy. WSL copy deleted. Rule: only ever work from PS C:\dev\intern>, never a wsl.localhost path. Partially resolves the "target OS first" open question (committed to Windows-first dev).

- **Conversation persistence + /resume (store plugin, no Rust).** Conversations are serialized to `sessions.json` via `tauri-plugin-store`, keyed by id, with a `currentId` pointer (`src/session.ts`). The current session restores on launch and saves on every change; empty conversations are never written and no-op renders are skipped via a last-saved snapshot so merely relaunching does not bump a session's timestamp. Slash input (`/resume`, `/new`) routes through a small command registry in `App.tsx` instead of Claude, so new commands are one map entry. The `/resume` list renders inline in the stream (no modal, per the app's pattern). CRITICAL invariant: every conversation switch (new, resume, load-a-row) calls `persistCurrent()` BEFORE clearing the view, so in-progress work is never lost. Titles: generated by ONE cheap Haiku call (`generateTitle`, no tools, no cached system prompt, `max_tokens: 16`, only trimmed opening turns sent) once a conversation hits 3+ messages, cached on the session and never regenerated; under 3 messages stays on the first-message placeholder so throwaway chats do not clutter the list or cost anything. Race guarded: the async title-save writes the LATEST messages (via `messagesRef`), not the stale snapshot that triggered generation, so a reply landing mid-request is not clobbered.

## Blockers / Gotchas

- **Azure app-registration saga (AADSTS70000 incremental consent → audience mismatch).** Adding Mail drafting required Mail.ReadWrite, which set off a chain of auth problems worth recording so we never repeat it.
  - **Stranded original app.** The original registration (client ID `c44b4083` per Jonah's Azure notes; the ID that had been in the codebase was `233ef4ee`, swapped out in commit history) lived in the **Comcast default-directory tenant**, which no accessible account could edit. So Mail.ReadWrite could not be added to it. Consenting to a scope the app doesn't expose caused **AADSTS70000** on the next token refresh (the refresh_token was for a scope set the app couldn't grant).
  - **Incremental-consent trap.** With `prompt=select_account`, reconnecting reused cached consent and silently skipped granting the newly-added scope. `prompt=consent` forces the screen and re-grants, but that was a workaround, not the fix.
  - **Real fix: fresh registration under the personal account.** Registered a new app (`f4c142b4-85f2-4d7d-801d-5e8c59fe0676`) under the personal outlook.com account (`jonah.schwartz5`) via entra.microsoft.com, with all four Graph scopes at registration (`Calendars.ReadWrite`, `Mail.ReadWrite`, `User.Read`, `offline_access`), redirect `http://localhost` under Mobile/desktop, and "Allow public client flows" = Yes (pure PKCE, no secret).
  - **Audience mismatch: `/consumers/` not `/common/`.** The new app is **Personal-accounts-only** (Consumer audience). `/common/` requires an "All accounts" audience and fails with an audience mismatch. Switched `AUTHORITY` in msauth.ts to `https://login.microsoftonline.com/consumers/oauth2/v2.0`. Reverted `prompt` back to `select_account` since the fresh app has both scopes from the start (no forced consent needed).
  - **Lessons:** register the Azure app under the personal MS account that actually owns the calendar/mail (not an org/Comcast default tenant); add ALL needed Graph scopes at registration up front (avoids AADSTS70000 incremental-consent pain); match the auth endpoint to the app's audience (`/consumers/` for personal-only, `/common/` only for multi-audience apps).
- **AADSTS90023 on the Microsoft token exchange (Origin header).** The `/token` call failed with AADSTS90023 (cross-origin token redemption) even though it goes through `@tauri-apps/plugin-http` (Rust-side, not browser fetch). Cause: plugin-http's Rust `commands.rs` auto-injects an `Origin` header derived from the webview URL (`http://localhost:1420` in dev, `tauri://localhost` in prod). Microsoft rejects any token request carrying Origin unless the app is registered as an SPA. Fix, two parts, BOTH required: (1) build the crate with the feature: `tauri-plugin-http = { version = "2", features = ["unsafe-headers"] }`; (2) send `Origin: ""` in the request headers. The plugin only strips Origin when unsafe-headers is on AND Origin is explicitly empty. `Origin: ""` survives the JS `new Headers(obj)` path because a directly-constructed Headers has guard "none" (forbidden-header filtering only applies to Headers taken from a Request). Both token calls use the shared `TOKEN_HEADERS` const in msauth.ts. Apply the same `Origin: ""` to future Graph calls if any endpoint objects to it.
